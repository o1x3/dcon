# Code of Conduct

## Our pledge

We want dcon to be a friendly, harassment-free project for everyone, regardless
of experience, background, or identity.

## Standards

Examples of behaviour that helps:

- Being respectful and constructive in issues, PRs, and reviews.
- Welcoming newcomers and answering "why does Docker do X" questions kindly.
- Focusing on what's best for users of the tool.

Unacceptable behaviour includes harassment, personal attacks, trolling, and
publishing others' private information.

## Scope & enforcement

This applies in all project spaces (issues, PRs, discussions) and when
representing the project publicly. Report unacceptable behaviour to the
maintainer via a GitHub private message or a security advisory; reports will be
reviewed and handled confidentially.

This Code of Conduct is adapted from the
[Contributor Covenant](https://www.contributor-covenant.org), version 2.1.
